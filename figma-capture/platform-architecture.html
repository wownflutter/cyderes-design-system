Cyderes Platform Architecture Proposal

Internal UX Architecture — Unified Navigation, Page Flows, and Security Graph Model

March 2026

This document defines the unified architecture for the Cyderes platform spanning MDR (Detect & Respond), Exposure Management, Asset Management, and Identity & Access Management. All four domains are connected by the Cyderes security graph — a shared data model that enables cross-domain investigation, contextual navigation, and a single pane of glass for security operations. The architecture herein covers the application shell, navigation hierarchy, page templates, page flows per domain, the security graph object model, and the cross-domain pivot model that makes the platform feel like one continuous tool rather than four bolted-on products.

The graph is not a visualization — it's a data model

Every object in the platform (asset, identity, exposure, detection) is a node. Every relationship (accesses, has, fires on) is an edge. The Issue Map renders this model spatially.

Cross-domain connections are first-class

An identity accesses an asset that has an exposure that enables an attack path that triggers a detection. This chain spans 4 domains. The graph makes it one continuous investigation.

The graph drives navigation

Clicking a node navigates to that object's detail page. The analyst moves between domains by following relationships, not by switching tools. Breadcrumbs record the journey.
